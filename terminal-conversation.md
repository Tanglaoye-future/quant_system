  用户目标                                                                                                                                                                                                                                                                                                  
                                                                                                                                                                                                                                                                                                              - 市场：A 股 + 港股                                                                                                                                                                                                                                                                                       
  - 频率：20-60 个交易日（中线）                                                                                                                                                                                                                                                                            
  - 方法：自上而下 + 自下而上选股 → 催化剂识别 → 技术择时 → 动态风控 → 单笔交易归因复盘
  - 执行：手动下单（券商：东方财富）

  关键事实澄清

  1. gs-quant ≠ 高盛交易框架：高盛 SecDB/Slang 不开源，gs-quant 只是开源分析库（pricing / risk / timeseries），不是策略框架
  2. 东方财富自动化：A 股只能走 QMT + xtquant（50 万门槛），港股无自动化通道；用户选了手动下单，绕开此问题
  3. FinceptTerminal 价值有限：欧美数据为主，无大陆券商，AGPL 协议有传染性。结论是不绑死该终端，独立 Python 项目更合适
